    WPS LPR Concentrador

    Pré-Requisitos

    • ParkingPlus versão mínima: 7.364.1
    • Serviço Concentrador de LPR na mesma versão do ParkingPlus.
    • Câmeras homologadas até a última atualização do manual:
      • Pumatronix Leitura Contínua
      • Harpia
      • Harpia TCP

    Modelo Conceitual e Funcionamento

    Modelo Conceitual

    Funcionamento

    A integração funciona da seguinte forma:

    1. O Serviço Concentrador deverá ser instalado em um servidor ou computador na rede da garagem de forma que o ParkingPlus Caixa ou Terminal enxergue o serviço e o serviço enxergue as câmeras.
    2. As câmeras que o terminal desejar receber leituras deverão ser cadastradas no módulo específico que é explicado na seção Configurações deste manual.
    3. Ao Iniciar o Terminal, o sistema irá cadastrar no serviço as câmeras que estão cadastradas no mesmo, e em seguida irá inscrever o terminal no serviço para que ele consiga receber as leituras que o serviço concentrador realizar.
    4. Após isso, em cada leitura que o serviço receber das câmeras, as mesmas serão enviadas para o terminal tratar da forma que bem entender
    5. Neste processo as leituras são acumulados em um processo que chamamos de AMOSTRAGEM de leituras.
      • Este processo irá receber as leituras e guardá-las em um cache de memória no terminal
      • Ao guardar neste cache, automaticamente por meio de um algorítmo interno no terminal, o sistema tenta identificar se ainda é uma leitura do mesmo acesso ou se é uma nova leitura
      • Além disso o Algorítmo tentará identificar qual a melhor leitura desse referido acesso
      • Essa melhor leitura fica aguardando o terminal solicitar a placa de acesso
    6. No processo de acesso, por padrão o terminal dispara TRIGGERS para obter a leitura atual na câmera, porém com a LPR WPS esse Trigger não busca mais a leitura na câmera, e sim no processo de amostragem explicado anteriormente, retornando apenas a melhor leitura do acesso que já estava identificada conforme recebemos as leituras do concentrador.
      • Dessa forma tente a ficar rápido o acesso, pois a leitura já estará disponível quando o terminal precisar.

    Configurações

    Serviço Concentrador LPR

    É obrigatório que o serviço concentrador LPR esteja instalado na mesma rede da garagem para que essa funcionalidade aconteça.

    O ParkingPlus se inscreve no serviço para receber as leituras das câmeras. O serviço por sua vez se comunica com a câmera e a cada leitura realizada pela mesma, o serviço repassa para os inscritos, no caso desta funcionalidade, o ParkingPlus Terminal.

    Instalação

    O serviço concentrador LPR deve ser instalado seguindo sempre a mesma versão de ParkingPlus da garagem, e possui versões Linux (preferencial) e Windows.

    Linux

    Encontrado no seguinte caminho de cada release do ParkingPlus:

    ...ParkingPlus_v7.XXX.X\Product\Internal\Sources\build\releases\ParkingPlus-7.XXX.X\Linux\WpsLprConcentrator\Instalacao\wpslprconcentratorservice-7.XXX.X-1.noarch.rpm

    Onde 7.XXX.X representa a versão do Release do ParkingPlus.

    Para instalar o rpm, execute:

    yum localinstall wpslprconcentratorservice-7.XXX.X-1.noarch.rpm

    Consulte o status do serviço para ter certeza do sucesso da instalação:

    systemctl status WpsLprConcentrator.service
● WpsLprConcentrator.service - wpslprconcentratorservice
   Loaded: loaded (/usr/lib/systemd/system/WpsLprConcentrator.service; disabled; vendor preset: disabled)
   Active: inactive (dead)
    Windows

    Importante

    Somente utilize o serviço windows quando não for possível sua instalação no LINUX

    Encontrado no seguinte caminho de cada release do ParkingPlus:

    ...ParkingPlus_v7.XXX.X\Product\Internal\Sources\build\releases\ParkingPlus-7.XXX.X\Windows\WpsLprConcentrator\Instalacao

    Onde 7.XXX.X representa a versão do Release do ParkingPlus.

    O diretório Instalação deverá ser copiado inteiro em uma pasta qualquer.

    Renomeie o arquivo wpslprconcentrator.sample.properties para wpslprconcentrator.properties e o configure (veja a próxima seção deste manual)

    Este arquivo deverá estar dentro de uma pasta na raiz com o seguinte caminho:

    \etc\WpsLprConcentrator\

    Configuração

    Arquivo: wpslprconcentrator.properties.

    Exemplo:

    ## Web Server Container Configurations ##
webserver.port = 3333

## General Config ##
config.snapshot.port=5000
config.rest.base_path=lprconcentratorwps/lpr
config.persist_local.subscriptions=false
config.persist_local.path=/etc/WpsLprConcentrator
  ## Windows Example: C\:\\etc\\WpsLprConcentrator
  ## Linux Example: /etc/WpsLprConcentrator

## Specific config to Pumatronix CR Driver
config.pumatronix_cr.photo_quantity = 1
config.pumatronix_cr.read_timeout = 1000
config.pumatronix_cr.get_plate_timeout = 1000

    onde:

    • webserver.port: É a porta que o serviço irá rodar no servidor

    • config.snapshot.port: É a porta que será utilizada para buscar um snapshot de câmeras WpsLpr.

    • config.rest.base_path: É o endpoint base do serviço, procure não alterar para não correr o risco de invalidar as inscrições

    • config.persist_local.subscriptions: Se true indica que o serviço irá persistir localmente uma configuração de inscrições realizadas no mesmo. É útil para casos que o serviço reiniciou sozinho e com isso nao perder as inscrições que estavam ativas no momento do crash.

    • config.persist_local.path: Local onde irá salvar as inscrições ativas se a config anterior estiver true.

    • config.pumatronix_cr.photo_quantity: Configuração específica para câmera pumatronix. Indica quantas fotos cada leitura obtida irá retornar. Padrão/Aconselhável = 1

    • config.pumatronix_cr.read_timeout = Configuração específica para câmera pumatronix. Indica de quanto em quanto tempo (milesegundos) será perguntado pumatronix se existe uma nova leitura. Padrão/Aconselhável = 500

    • config.pumatronix_cr.get_plate_timeout = Configuração específica para câmera pumatronix. Indica de quanto em quanto tempo (milesegundos) poderemos tentar obter uma nova leitura após uma ultima recebida. Deve ser maior ou igual que a config anterior. Padrão/Aconselhável = 1000

    Linux

    Execute os comandos abaixo para garantir que o serviço será inicializado com o sistema e checar se as configurações estão corretas:

    systemctl enable WpsLprConcentrator.service
systemctl start WpsLprConcentrator.service
journalctl -u WpsLprConcentrator.service

    Observe os logs do journalctl procurando erros. Se não houver, o serviço está funcionando corretamente.

    Windows

    Execute o arquivo install.bat, será registrado e iniciado o serviço. Veja em services.msc se o Serviço tá listado e em execução. Os logs do serviço estarão em uma pasta na raiz chamada /logs

    Cadastro e configuração de Câmeras

    O cadastro das câmeras do concentrador LPR WPS é feito no menu:

    Configurações → Monitor → Concentrador LPR - Cadastro de Câmeras

    Permissões de usuário

    Através do Manager, pode-se permitir ou bloquer o acesso ao cadastro das câmeras:

    Exemplos

    Pumatronix

    Onde:

    • Descrição câmera: Nome único no sistema para identificar as configurações da Câmera.
    • Ativar leitura: Quando ativo o caixa passará a receber as leituras de placas desta câmera para que ele trate e valide se a mesma é apta para tentar a entrada automática.
    • Tipo da câmera: Somente câmeras homologadas serão listadas neste campo. É importante que a câmera esteja configurada para trabalhar no modo de leitura continua. Consulte o manual especifico para configurar a mesma.
    • Eficiência leitura: Independente da eficiência configurado na câmera, é possível especificar uma eficiência mais restrita neste campo. Então independente de receber uma leitura, se a eficiência informada aqui for superior a recebida, o sistema irá ignorar a leitura.
    • URL do Serviço Concentrador LPR: É o caminho que esta rodando o serviço explanado no tópico anterior. Ao clicar em INCLUIR o sistema já irá preencher um caminho padrão, ajuste apenas o IP e porta do mesmo.
    • Porta callback: é a porta no caixa ou terminal que o serviço concentrador usará para enviar as leituras realizadas. Procure usar a mesma porta para todas as câmeras que cadastrar no caixa ou terminal.
    • Tentativas conexão: é a quantidade de tentativas que o concentrador de LPR fará para enviar uma leitura realizada. Após a quantidade informada neste campo ser alcançada sem sucesso no envio pelo concentrador, o mesmo irá parar o envio de leituras para a porta configurada no callback. Quando isso acontecer o caixa ou terminal deverá ser reaberto para a inscrição de recebimento acontecer novamente. Deixe ZERO para não ter esse limite.
    • Receber imagem na leitura: Quando ativo as leituras de placas realizadas pelo concentrador serão enviadas com a imagem da placa capturada para serem salvas em banco de dados.
      • Se ainda desejar que a imagem seja salva na tabela ImagensLpr será necessário configurar no caixa ou terminal a configuração avançada:
        • Seção: LPR
        • Chave: Registrar Imagens
    • CFG. CÂMERA: São as configurações específicas que cada câmera informada no campo TIPO DA CÂMERA possuem.
    • CFG. AVANÇADAS: São as configurações avançadas de leitura da câmera, onde:
      • Invervalo para nova leitura: Tempo em segundos que o sistema utiliza para considerar como uma nova leitura(novo acesso) a leitura atual.
      • Máxima diferença entre placas: Quantidade de caracteres diferentes que o sistema considera uma mesma placa.
      • Tempo de vida da leitura: Tempo em segundos que o terminal considera uma leitura existente no cache válida.
      • Máximo de leituras em cache: Quantidade de leituras a serem mantidas em cache simultaneamente. Novas leituras não serão adicionadas até que as antigas sejam invalidadas.
      • Timeout Lpr para Informatizado: Tempo em segundos que o sistema irá considerar que uma placa que realizou a saída do sistema poderá entrar novamente de forma automática na garagem. Somente para o Terminal e Caixa Informatizado.

    Para cadastrar basta clicar em INCLUIR e preencher os campos. Finalizando clique em CONFIRMAR. 

    Para Alterar basta selecionar uma câmera já cadastrada no box APELIDO DA CÂMERA e depois clicar no botão alterar. Altere os valores desejados e Finalizando clique em CONFIRMAR.

    Para Excluir basta selecionar uma câmera já cadastrada no box APELIDO DA CÂMERA e depois clicar no botão Excluir. Confirme a exclusão.

    Quando uma câmera for salva, alterada ou excluída do sistema será necessário fechar e abrir o caixa ou terminal novamente para que as alterações surtam efeito.

    Atenção

    Se a chave ATIVAR RECEBIMENTO DE LEITURA DESTA CÂMERA estiver desativada para todas as câmeras do terminal, o sistema irá trabalhar com o modelo antigo de LPR(se ativo), configuradas exclusivamente na seção de LPR dos TERMINAIS.

    Se a chave RECEBER IMAGEM NA LEITURA estiver ativa, será salva a imagem do momento do acesso.

    Harpia

    Para configurar uma câmera LPR Harpia, basta preencher o formulário como mostra a seção anterior, porém, no campo Tipo de câmera, deve-se escolher Harpia, e no grid Configuração da câmera, neste mesmo formulário, é preciso se atentar à alguns campos:

    onde:

    • Tópico MQTT: Este campo é obrigatório, e seu valor corresponde ao tópico no qual a câmera LPR está configurada, e vai publicar os dados de leitura;
    • Host MQTT: Este campo é obrigatório, e seu valor corresponde ao endereço do Broker MQTT. Como no exemplo acima, é necessário que seja definido o protocolo no qual o host está configurado (na maioria das vezes tcp://);
    • Porta MQTT: Este campo também é obrigatório, e seu valor corresponde a porta onde o Broker MQTT está disponível geralmente seu valor será 1883;
    • Usuário de Acesso: Este campo é opcional, e será usado quando o Broker MQTT for privado e precise de credencial para acesso. Seu valor corresponde ao usuário que tem permissão para acessar o Broker MQTT;
    • Senha de Acesso: Este campo é opcional, e será usado quando o Broker MQTT for privado e precise de credencial para acesso. Seu valor corresponde à senha do usuário que tem permissão para acessar o Broker MQTT;
    Harpia - TCP

    Para configurar uma câmera LPR Harpia TCP, basta preencher o formulário como mostra a seção anterior, porém, no campo Tipo de câmera, deve-se escolher Harpia - TCP, e no grid Configuração da câmera, neste mesmo formulário, é preciso se atentar à alguns campos:

    onde:

    • IP da Câmera: Este campo é obrigatório, e seu valor corresponde ao endereço IP da câmera;
    • Porta: Este campo é obrigatório, e seu valor corresponde a porta TCP da câmera;
    • Timeout - Keep Alive: Este campo também é obrigatório, e seu valor corresponde ao tempo em segundos que o concentrador deve esperar por uma resposta da câmera;

    Terminal

    Mesmo configurando a câmera na tela de cadastro exclusiva do Concentrador LPR WPS, algumas configurações avançadas de LPR seguem na tela de cadastro do Terminal e nas configurações avançadas.

    Atenção

    Algumas propriedades estarão desabilitadas quando ativo o recebimento de leituras via LPR WPS. Se o Checkbox "Ativar LPR" do terminal for marcado(ativo) com o concentrador ativo no mesmo terminal, o próprio terminal irá passar a utilizar o LPR integrado ao invés do concentrador.

    As configurações seguem com o mesmo objetivo de quando é uma câmera LPR no padrão antigo.
