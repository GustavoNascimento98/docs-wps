    PPRegistraSetor - Serviço para mudança de setor

    Informações gerais

    Serviço desenvolvido para realizar o processo de mudança de setor de um veículo que está na garagem. Deverão ser informados a placa, que deve ser uma placa válida que está no patio e o setor de destino, que deve existir na lista de setores cadastrados no WPS. É possível também atribuir uma tarifa específica, informando o id de uma tarifa existente no sistema WPS.

    Integração com PARKEYES - na inicialização do serviço e a cada mudança de dia, carrega uma lista de vagas e seus respectivos setores e depois realiza polling com temporização configurável das últimas atualizações das vagas, informando id da vaga e placa do veículo que a está ocupando.

    Por este serviço também é realizada a mudança de setor através da comunicação de LPRs com o WPS LPR Concentrador - o serviço carrega uma lib que encapsula todo o tratamento da comunicação das LPRs com o concentrador e efetiva a mudança de setor no sistema WPS.

    Instalação

    O serviço de mudança de setor deve ser instalado seguindo sempre a mesma versão de ParkingPlus da garagem.

    Ele pode ser encontrado no seguinte caminho de cada release do ParkingPlus:

    ...ParkingPlus_v7.XXX.X\Product\Internal\Sources\build\releases\ParkingPlus-7.XXX.X\Linux\PPRegistraSetor\Instalacao\servicoregistrasetor-7.XXX.X-1.noarch.rpm

    Onde 7.XXX.X representa a versão do Release do ParkingPlus.

    Para instalar o rpm, execute:

    yum localinstall servicoregistrasetor-7.XXX.X-1.noarch.rpm

    Consulte o status do serviço para ter certeza do sucesso da instalação:

    systemctl status PPRegistraSetor.service
● PPRegistraSetor.service - PPRegistraSetor
   Loaded: loaded (/usr/lib/systemd/system/PPRegistraSetor.service; disabled; vendor preset: disabled)
   Active: inactive (dead)

    Configuração do serviço

    O serviço deve ser configurado pelo arquivo "registrasetor.properties", que deve estar localizado na pasta ../etc/PPRegistraSetor.

    Exemplo de conteúdo do arquivo registrasetor.properties:

    # URL JDBC para conexão ao banco de dados.
javax.persistence.jdbc.url=jdbc:postgresql://192.168.0.197:5432/parkingplus
javax.persistence.jdbc.user=postgres
javax.persistence.jdbc.password=postgres

# URL base para WS
config.ws.base_url=http://HOST:PORT/IMAGiNAPERest/v1.0
config.ws.timeout=10
config.time_busca_vagas=120
config.diferenca_placa=1
config.permite_apenas_uma_mudanca_setor=true
config.integracao_parkeyes=true
config.atualiza_com_setor_padrao=false
config.setor_padrao=ESTACIONAMENTO
config.tolerancia_minima=1
config.mudanca_setor_com_concentrador=false

## Configuracao dos Schedulers - CRON ##
scheduler.pool_size = 1
scheduler.test = 0/10 * * * * *
scheduler.atualizasetor = */30 * * * * *
    Configuração Descrição
    javax.persistence.jdbc.url Path arquivo banco de dados WPS Parking Plus.
    javax.persistence.jdbc.user Usuário banco de dados WPS Parking Plus.
    javax.persistence.jdbc.password Senha banco de dados WPS Parking Plus.
    config.ws.base_url URL base para WebService do integrador.
    config.ws.timeout Tempo em segundos para timeout com Webservice do integrador.
    config.time_busca_vagas [PARKEYES] Tempo em segundos para ser enviado como parâmetro do métodos de busca de atualizações das vagas.
    config.diferenca_placa Máximo de quantidade de caracteres diferentes referente à placa lida e a placa que está no pátio.
    config.permite_apenas_uma_mudanca_setor Indica se permite apenas uma mudança de setor para cada veículo. Se ocorrer a solicitação para o mesmo veículo mudar de setor novamente, não será realizada.
    config.integracao_parkeyes Ativa ou não integração com PARKEYES.
    config.atualiza_com_setor_padrao Indica se atualiza com setor padrão (indicado na configuração config.setor_padrao) no caso da integração não retornar nenhum setor. Desativado por padrão.
    config.setor_padrao Setor padrão para realizar a mudança de setor, caso nenhum setor seja informado na integração.
    config.tolerancia_minima Tempo em minutos de tolerância mínima para realizar a mudança de setor.
    config.mudanca_setor_com_concentrador Ativa a mudança de setor via WPS LPR Concentrador e LPRs.
    scheduler.pool_size Set the pool of threads.
    scheduler.test Tempo em segundos para rodar a rotina de testes (esta rotina loga a mensagem: " Teste de Scheduler") => no exemplo acima, está configurado para a cada 10 segundos.
    scheduler.atualizasetor Tempo em segundos para rodar a rotina para atualização de setor => no exemplo acima, está configurado para a cada 30 segundos.

    Integração com PARKEYES

    Algumas particularidades em relação à integração com Parkeyes:

    a. integração via WebService

    b. na inicialização do serviço e a cada mudança de dia, carrega uma lista de vagas e seus respectivos setores e realiza polling com temporização configurável das últimas atualizações das vagas, informando id da vaga e placa do veículo que a está ocupando.

    c. métodos utilizados:

    /general/spaces => utilizado para pegar a lista de vagas e seus respectivos setores - sinalizado no campo "tariffzone"

    /parkingSpaces/status/x => utilizado para pegar as últimas atualizações de ocupação das vagas dos últimos "x" segundos ("x" configurável em config.time_busca_vagas)

    Mudança de setor via LPR e WPS LPR Concentrador

    Neste modo de integração, o serviço carrega e inicia uma lib (PPDLLMudaSetor.dll para Windows ou libPPDLLMudaSetor.so para Linux) que encapsula toda a parte de comunicação com o WPS LPR Concentrador. Esta lib verifica lista de terminais de passagem virtuais configurados no sistema WPS e gerencia a comunicação das LPRs configuradas para cada terminal e o WPS LPR Concentrador.

    Atenção

    Apenas salientando que o WPS LPR Concentrador deve estar executando, esperando as informações das LPRs e repassando para o serviço. Mais informações sobre instalação e configuração do WPS LPR Concentrador aqui.

    b. Ainda pelo ParkingPlus Manager, também devem estar configuradas as LPRs para cada terminal através do menu "Configurações" → "Monitor" → "Concentrador LPR - Cadastro de Câmeras":

    c. Para inicialização da lib, deve existir o arquivo de configuração para indicar o banco de dados da garagem:

    - no Windows: %homedrive%%homepath%\PPDLLMudaSetor.ini
- no Linux: /etc/PPDLLMudaSetor.conf
- conteúdo do arquivo:

    [BANCODADOS]
    TipoBD=0
    Banco=192.168.0.197:parkingplus
    Port=5432
    QtdeBD=0

    d. No arquivo de configuração do serviço, habilitar esta integração pela chave abaixo:

    config.mudanca_setor_com_concentrador=true
