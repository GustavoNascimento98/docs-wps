    Tap&GO

    O Tap&GO é o nome interno para entradas, pagamentos e saídas realizadas com cartões de crédito / débito via aproximação.

    Requisitos

    Para que o Tap&GO funcione os terminais (tanto entrada quanto saída) devem:

    • Possuir um leitor específico, homologado pela operadora do cartão.
    • Utilizar as imagens de boot via HTTP. Isto exclui terminais LX800, que não suportam boot HTTP.
    • Possuir conectividade com um servidor de pagamento (PSP) fornecido pela operadora dos cartões.

    Funcionamento

    Com o Tap&GO ativado usuários de cartões de crédito / débito com suporte à proximidade podem entrar no estacionamento usando o cartão como identificação.

    Clientes que acessam pelo Tap&GO são rotativos normais. Não existe ticket impresso, mas fora isto não há diferença entre usuários Tap&GO e rotativos. Se o terminal tiver LPR, o cliente pode ser localizado no pátio pela placa.

    Entrada

    Com o Tap&GO ativado, enquanto houver presença no terminal, o cliente pode aproximar o cartão ao leitor para entrar.

    Se o cartão for compatível com o leitor e for da operadora correta, o terminal vai autorizar a entrada e abrir a cancela sem imprimir ticket.

    Mude a mensagem de presença do terminal para indicar ao cliente que esta opção existe, caso desejado.

    Pagamento

    Exceto em garagens com LPR, não existe como localizar o cliente no pátio para fazer o pagamento.

    O pagamento pode ser feito pelo próprio cartão no terminal de saída.

    Saída

    Para sair, o cliente deve usar o mesmo cartão da entrada.

    São necessários 2 taps; o primeiro para localizar o ticket no pátio e calcular o valor devido, e o segundo tap para fazer a transação de pagamento.

    Também entre os taps, caso assim configurado, o terminal consulta uma API da operadora do cartão para saber se o cliente tem direito a desconto, e já aplica o valor informado na transação de pagamento.

    Todos os valores, inclusive do desconto, estão disponíveis para uso nas mensagens do terminal.

    Em caso de saída via LPR (e.g Ticketless), pode ser que apenas o tap de pagamento seja necessário. Nestes casos, o desconto ainda pode ser aplicado se os dados necessários do cartão foram coletados na entrada. O Parking Plus não têm como controlar se a leitora envia estes dados na entrada ou não.

    Pagamento de Avulsos

    Além de permitir o acesso de clientes através do seu cartão bancário, o Tap&GO também pode ser usado para pagamentos de tickets comuns.

    Com esta função ativada o Tap&Go será habilitado para pagamento em qualquer evento de ticket não pago, seja o acesso por Tap ou não.

    Caso o cliente não possua / deseje utilizar o Tap&GO para pagar o acesso o terminal só vai abrir o retorno após o timeout para pagamento. Isto pode causar fila e transtornos, portanto esta funcionalidade não é habilitada por padrão. Para ativa-la utilize: TAP&GO/SOLICITAR_TAP_PARA_PAGAR_TICKET_IMPRESSO_NAO_PAGO.

    Configuração

    O Tap&GO não depende de nenhum serviço do Parking Plus; todas as configurações serão feitas nos terminais, individualmente.

    A maior parte destes valores deve se fornecida pela operadora do cartão.

    Configurações Necessárias

    Configurações que podem ser alteradas livremente para ajustar o Tap&GO aos desejos da garagem.

    String

    Descrição

    String em JSON contendo os dados que devem ser enviados ao daemon.

    Números geralmente devem estar entre aspas, ou vão ser transformados em decimais (e.g 10 vira 10.00).

    O Formato e conteúdo desta configuração deve ser fornecido pela operadora de cartões. Esta configuração vai ser diferente em cada terminal.

    TAP&GO

    SOLICITAR_TAP_PARA_PAGAR_TICKET_IMPRESSO_NAO_PAGO

    Boolean

    0

    Descrição

    Ativa ou não pagamento com Tap&GO para avulsos com ticket.

    Apesar do nome, esta chave afeta solicitação de pagamento para todos os avulsos, sejam com ticket impresso ou Ticketless!

    Transtorno Operacional

    Com esta função ativada, todos os clientes com ticket não pago na saída vão ter que aguardar o timeout de pagamento do Tap&GO antes da abertura do retorno. Para mais detalhes, veja a explicação.

    Seção

    Chave

    Tipo

    Padrão

    #

    TAP&GO

    EMV_TYPE_TAP_PAGAMENTO

    String

    Descrição

    Indica, no Tap para identificação do cliente, qual a preferência de modalidade do cartão (em caso de cartão multiplo).

    • 0: Deixa o cartão/leitor negociarem qual a modalidade preferida e tenta pagar apenas pela modalidade negociada. Este é o padrão.
    • 1: Aceita exclusivamente crédito. Transação negada se não for possivel crédito.
    • 2: Aceita exclusivamente débito. Transação negada se não for possivel débito.
    • C: Modo combo. Tenta a transação com ambos os modos do cartão, e só nega a transação se ambos os modos forem negados (o daemon não diz qual modo será tentado primeiro).

    Seção

    Chave

    Tipo

    Padrão

    #

    TAP&GO

    EMV_TYPE_TAP_ID

    String

    Descrição

    Idem ao TAP&GO/EMV_TYPE_TAP_PAGAMENTO, porem se aplica ao tap inicial para identificação.

    Util para situações nas quais a entrada precise fazer uma transação de caução, por exemplo.

    Configurações Internas

    Estas configurações geralmente não devem ser alteradas. Elas existem para lidar com alterações de última hora por parte da operadora. Só as altere se for solicitado por um desenvolvedor.

    Verifique os padrões

    Se você tiver problemas com o Tap&GO, verifique se estas configurações não foram alteradas. Caso tenham sido, informe o desenvolvimento do problema e da alteração. Vamos confirmar se a alteração foi devida ou indevida.

    Esta configuração existe para uso interno ou casos especiais. Não altere esta configuração sem orientação direta.
