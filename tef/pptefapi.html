    PPTEFAPI - Serviço de TEF

    Descrição

    Neste manual iremos abordar como rodar o serviço PPTEFAPI, o serviço é um GATEWAY entre as aplicações da garagem e os serviços de pagamento, sendo um intermediário.

    Pré-Requisitos

    • ParkingPlus versão mínima: 7.454.0
    • Serviço do PPTEF na mesma versão do ParkingPlus.
    • Centos 7 ou superior, Se for outra distro linux só garantir que o mesmo rode o docker.

    Configurações

    A pasta em que se pode configurar o serviço vai estar em /etc/ParkingPlus/PPTEFAPI

    Podendo alternar entre o ambiente de desenvolvimento e produção tanto quanto a porta que a aplicação está rodando.

    Instalação

    O serviço PPTEFAPI deve ser instalado seguindo sempre a mesma versão de ParkingPlus da garagem.

    O PPTEFAPI é uma aplicação JAVA 21 que roda dentro de um container Docker, para isso será necessário fazer a instalação padrão para docker com o pkplus-cli.

    Properties docker

    Chave Padrão Objetivo
    prod_enviroment true Indicar se o ambiente é produção ou não, o padrão é sempre produção.
    request_sync false Indica qual o modo de operação do PPTEFAPI, o mesmo deve estar no mesmo modo de operação da garagem.
    url_live_checkout_prefixo urldecheckoutprefixoparaprod Indica qual é o prefixo da URL de checkout, se utilizar a padrão ou um prefixo incorreto os pagamentos ficaram sem confirmação IMPORTANTE SE ATENTAR
    main.jettyPort 8980 Será a porta que a aplicação irá subir, só altere se houver conflito de portas.
