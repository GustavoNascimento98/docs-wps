    Integração Conectcar Informatizado

    Versão mínima

    Este manual foi desenvolvido para ser aplicado na versão 7.280.0 e futuras versões. Versões anteriores a esta não contemplam esta funcionalidade.

    Este guia explica em detalhes como proceder para a instalação, configuração e ativação da funcionalidade de acesso informatizado com pagamento pelo ConectCar. A instalação pode incluir um serviço a ser instalado em um Tomcat no servidor da garagem dependendo dos tipos de estações presentes no sistema.

    Instalação - Procedimentos

    1) Instalar o serviço de gateway no servidor

    Importante

    No caso da ConectCar, a instalação do serviço de gateway é OBRIGATÓRIO, e é o mesmo da integradora Sem Parar, chamada gatewayacessowebsemparar.war. Deve ser utilizada com a versão 7.280.0 ou superior, equivalente à versão do software a ser instalado.

    O serviço de gateway serve de intermediário entre os caixas de boot remoto e o serviço de acesso do Sem Parar. Este serviço depende de ao menos uma instância de Tomcat instalada e configurada no servidor da garagem. A instalação do TomCat no servidor não será abordada neste manual. Refira-se a Servidor - Instalação do Tomcat para um guia neste assunto.

    Assumindo que existe uma instância de Tomcat na pasta /WPSBrasil/tomcat no servidor da garagem, basta parar o serviço do tomcat, copiar o pacote gatewayacessowebsemparar.war para a pasta /WPSBrasil/tomcat/webapps/ e reiniciar o tomcat.

    /WPSBrasil/tomcat/bin/shutdown.sh
cp /Instalacoes/gatewayacessowebsemparar.war /WPSBrasil/tomcat/webapps/
/WPSBrasil/tomcat/bin/startup.sh

    Depois, teste o serviço acessando a seguinte URL, relativa a instalação do tomcat no servidor: gatewayacessowebsemparar/servicos/gatewaySpvf/tokenForUrl?url=abc. Por exemplo, se a instalação do sistema for completamente padrão, a URL completa deverá ser: http://125.125.10.100:8080/gatewayacessowebsemparar/servicos/gatewaySpvf/tokenForUrl?url=abc. A resposta deve ser uma mensagem de erro (sim) igual ou muito parecida com esta:

    {"response":{"gatewaySuccess":false,"gatewayMessage":"java.net.MalformedURLException: no protocol: abc","payload":null}}

    2) Configurar os dados de acesso ao Web Service do ConectCar

    Para permitir acessos informatizados é necessário ativar a funcionalidade e dizer onde esta o serviço do ConectCar, bem como informar alguns códigos fornecidos pelo ConectCar necessário para se autenticar com o serviço. Esta configurações devem ser feitas em Configurações -> Monitor -> Serviços Tag Pag. Auto, conforme abaixo:

    Chave Objetivo
    api_inf__enable Indica se a funcionalidade deve ser ativada ou não, para todo o sistema
    api_inf__svc_cod_estabelecimento Código de Estabelecimento/Convenado informado pela ConectCar para autenticação. Muda de garagem para garagem e deve ser colocado da mesma forma que informado pela ConectCar.
    api_inf__svc_gateway_url URL do gateway local para acesso ao serviço da ConectCar em caixas Linux (boot remoto). Para a integradora ConectCar, ela é OBRIGATÓRIA. Neste caso é necessário instalar o serviço de gateway em um Tomcat no servidor da garagem. Use a URL de teste como base, removendo tudo após servicos/.
    api_inf__svc_endpoint_url URL do serviço da Conectcar, deve necessariamente conter o protocolo (http://, https://, etc). Pode-se confirmar se a URL é valida e acessível a inserindo-a em um navegador.
    api_inf__svc_username Usuário informado pela ConectCar para autenticação. Muda de garagem para garagem e deve ser colocado da mesma forma que informado pela ConectCar.
    api_inf__svc_password Senha informada pela ConectCar para autenticação. Muda de garagem para garagem e deve ser colocado da mesma forma que informado pela ConectCar.
    cx__preferir_pagamento_na_tag Caso deseja habilitar a preferência para realizar os pagamentos na tag, ativar esta chave ( valor = 1 )

    Cuidado

    Se o sistema da garagem estiver operando no momento da operação, certifique-se que todos os parâmetros estão corretos antes mudar api_inf__enable. Isto inclui instalar o serviço de gateway no servidor, se necessário.

    No caso da Conectcar, recomendamos reiniciar os aplicativos e caixa em caso de cadastro ou alteração dos dados da funcionalidade.

    3) Informatizado na EPA

    Para ativar o Informatizado na EPA, são pré-requisitos:

    • Sistema Parking Plus Manager atualizado e configurado;
      • Tela da EPA "Efetuar cobrança na Tag" devidamente ativa e configurada;
      • Sistema Parking Plus EPA e demais sistemas deverão estar devidamente atualizados e configurados para a versão mais recente;
    • Configurar para utilizar o serviço de gateway (gatewayacessowebsemparar);
    • O ticket/veículo precisa ter realizado com sucesso a pré-autorização de acesso via Tag3 Informatizado para que seja apresentada/possível cobrança via Tag Informatizada;

    Configurações Avançadas na EPA, seção TAG3_INFORMATIZADO:

    Chave Objetivo
    ATIVO Indica se a funcionalidade deve ser ativada ou não para a referida estação EPA.
    MENSAGEM TOKEN MANUAL Mensagem apresentada na tela da EPA, quando o tipo de Tag necessita de Token Externo de App de Terceiro.
    SEM_TOKEN_EXTERNO_PARA_VALOR_ZERADO No caso de Tag que necessita de Token Externo de App de Terceiro, ativar ou desativar solicitação de input de dados, para o caso de valor a ser cobrado for igual a 0 (zero).
    THREADS_NOTIFICACAO_BACKGROUND Quantidade de threads para executar em background o serviço de cancelamentos de pré-autorizações assíncronas (Se valor = 0, desativado para a estação)

    Obs.:

    • Talvez seja necessário reiniciar a(s) estação(ões) para aplicar as alterações realizadas;
    • Caso não apresente a tela "Efetuar cobrança na Tag" e o Manager esteja atualizado, pode ser necessário que o técnico limpe as configurações de tela, reinicie o Manager e depois tenha que refazer as configurações de tela da EPA novamente;

    Resultados e Orientações

    Após a configuração com sucesso da funcionalidade, os caixas que fizerem entrada no sistema também vão consultar a placa digitada junto ao ConectCar; Se for uma placa do ConectCar, a entrada será realizada como ConectCar Informatizado.

    Placas cuja entrada for feita desta forma serão cobradas no ConectCar, assim como entradas feitas por TAG. O Operador de caixa ainda pode optar por receber diretamente na garagem selecionando uma forma de pagamento manualmente no caixa (Tecla P), podendo ser inclusive dinheiro; Valores pagos desta forma não serão descontados da TAG mas ser for gerado excedente entre o pagamento e o momento da saída a diferença vai para a TAG.

    Observações

    Configure a geração dos logs do serviço do GatewayAcessoSemParar para gerar detalhamentos nos logs das transações/comunicações entre os sistemas da WPS (Parking Plus Caixa Entrada/Saída + GatewayAcessoSemParar) e ConectCar Informatizado (ws ConectCar)
