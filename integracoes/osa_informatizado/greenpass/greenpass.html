    Manual de instalação para GREENPASS informatizado

    Este guia contém informações de como proceder para a instalação, configuração e ativação da funcionalidade de acesso informatizado com pagamento pela GREENPASS. A instalação inclui um serviço a ser instalado em um Tomcat no servidor da garagem - gatewayacessowebsemparar.

    Instalação - Procedimentos

    Instalar o serviçao gatewayacessowebsemparar no servidor

    Importante

    No caso da GREENPASS, a instalação do serviço de gateway é OBRIGATÓRIO, e é o mesmo da integradora Sem Parar, chamada gatewayacessowebsemparar.war. Deve ser utilizada com a versão equivalente à versão do PKPLUS a ser instalado.

    OO serviço de gateway serve de intermediário entre os caixas/epas e o serviço de acesso da GREENPASS. Este serviço depende de ao menos uma instância de Tomcat instalada e configurada no servidor da garagem. A instalação do TomCat no servidor não será abordada neste manual. Refira-se a Servidor - Instalação do Tomcat para um guia neste assunto.

    Assumindo que existe uma instância de Tomcat na pasta /WPSBrasil/tomcat no servidor da garagem, basta parar o serviço do tomcat, copiar o pacote gatewayacessowebsemparar.war para a pasta /WPSBrasil/tomcat/webapps/ e reiniciar o tomcat.

    Depois, teste o serviço acessando a seguinte URL, relativa a instalação do tomcat no servidor: gatewayacessowebsemparar/servicos/gatewaySpvf/tokenForUrl?url=abc. Por exemplo, se a instalação do sistema for completamente padrão, a URL completa deverá ser: http://125.125.10.100:8080/gatewayacessowebsemparar/servicos/gatewaySpvf/tokenForUrl?url=abc. A resposta deve ser uma mensagem de erro (sim) igual ou muito parecida com esta:

    {"response":{"gatewaySuccess":false,"gatewayMessage":"java.net.MalformedURLException: no protocol: abc","payload":null}}

    Configurar os dados de acesso ao WebService GREENPASS

    Para permitir acessos informatizados é necessário ativar a funcionalidade e dizer onde está o serviço da GREENPASS, bem como informar alguns códigos fornecidos pela GREENPASS necessários para se autenticar com o serviço. Esta configurações devem ser feitas em Configurações -> Monitor -> Serviços Tag Pag. Auto -> GREENPASS, conforme abaixo:

    Chave Descrição
    api_inf__enable Indica funcionalidade ativada/desativada para GREENPASS. (1=ativada/ 0=desativada)
    api_inf__solicita_token_apos_pagamento Deve estar ativada para GREENPASS. O token é solicitado ao cliente após o envio das informações de pagamento para autorização deste. Obs.: O token é solicitado apenas quando valor maior que o configurado na chave api_inf__valor_maximo_sem_token.
    api_inf__solicita_token_por_valor Indica se o token será solicitado apenas se valor a pagar é maior que um limite retornado da integração. Para Greenpass, deixar habilitado (valor=1).
    api_inf__svc_client_id Código "ClientID" fornecido pela GREENPASS, utilizado para solicitar autenticação.
    api_inf__client_secret Código "ClientSecret" fornecido pela GREENPASS, utilizado para solicitar autenticação.
    api_inf__svc_cod_estabelecimento Código do Estabelecimento Comercial na GREENPASS (Estacionamento) fornecido pela GREENPASS.
    api_inf__svc_cod_terceiro (SCOPE) Código informado pela GREENPASS.
    api_inf__svc_endpoint_url URL base do serviço GREENPASS.
    api_inf__svc_gateway_url URL do gateway (gatewayacessowebsemparar) local para acesso ao serviço da GREENPASS. Item OBRIGATÓRIO. Neste caso é necessário instalar o serviço de gateway em um Tomcat no servidor da garagem.
    api_inf__svc_token_url URL para solicitação de token de autenticação.
    api_inf__utiliza_token Deve estar habilitada para solicitar token do cliente.
    cx__preferir_pagamento_na_tag Habilita a preferência para realizar os pagamentos na tag; por padrão, ela deve estar ativa. (1=ativado/0=desativado)
    cx__tempo_maximo_de_espera Tempo máximo para aguardar o retorno da situacao-transacao da GREENPASS.

    Informatizado na EPA

    Para ativar o Informatizado na EPA, são pré-requisitos:

    • Sistema Parking Plus Manager atualizado e configurado;
    • Tela da EPA "Efetuar cobrança na Tag" devidamente ativa e configurada;
    • Sistema Parking Plus EPA e demais sistemas deverão estar devidamente atualizados e configurados para a versão mais recente;
    • Configurar para utilizar o serviço de gateway (gatewayacessowebsemparar);
    • O ticket/veículo precisa ter realizado com sucesso a pré-autorização de acesso via Tag3 Informatizado para que seja apresentada/possível cobrança via Tag Informatizada;
    • Ativar configuração avançada na EPA, seção TAG3_INFORMATIZADO, chave ATIVO:

    Configurações Avançadas na EPA, seção TAG3_INFORMATIZADO:

    Chave Descrição
    ATIVO Indica se a funcionalidade deve ser ativada ou não para a referida estação EPA.
    MENSAGEM TOKEN MANUAL Mensagem apresentada na tela da EPA, quando o tipo de Tag necessita de Token Externo de App de Terceiro.
    SEM_TOKEN_EXTERNO_PARA_VALOR_ZERADO No caso de Tag que necessita de Token Externo de App de Terceiro, ativar ou desativar solicitação de input de dados, para o caso de valor a ser cobrado for igual a 0 (zero).
    THREADS_NOTIFICACAO_BACKGROUND Quantidade de threads para executar em background o serviço de cancelamentos de pré-autorizações assíncronas (Se valor = 0, desativado para a estação)

    Importante

    É necessário reiniciar as estações para aplicar as alterações realizadass.

    Observações

    Configure a geração dos logs do serviço do GatewayAcessoSemParar para gerar detalhamentos nos logs das transações/comunicações entre os sistemas da WPS (Parking Plus Caixa Entrada/Saída + GatewayAcessoSemParar) e GreenPass Informatizado (webservive GreenPass).
