    Login com LDAP

    Este guia explica em detalhes como proceder para a instalação, configuração e ativação da funcionalidade de Login com LDAP em um ambiente de Active Directory.

    Ao final do documento o Parkingplus Caixa, Manager e Papafila tentarão se logar na rede LDAP configurada e em caso de erro tentarão se logar pelo procedimento normal do sistema. Mais a frente teremos mais detalhes.

    Versão mínima

    Versão 7.448.x

    Este manual foi desenvolvido para ser aplicado e futuras versões. Versões anteriores a esta não contemplam esta funcionalidade.

    Ativação e Configuração

    1. Logue-se no PKPLUS Manager como SUPERUSER ou com um usuário que possua a permissão.

    2. Acesse o Menu: Configurações -> Monitor -> Active Directory - Config

    3. Opção de configurações, onde:

      • Ativar:
        • Ativa ou desativa a funcionalidade.
        • Quando desativado, as configs a seguir serão desconsideradas
      • Url Host LDAP:
        • É a URL do servidor onde esta rodando o serviço de Active Directory LDAP.
        • Essa informação deverá ser fornecida pelo Cliente
      • Porta Host LDAP:
        • Porta do serviço
        • Essa informação deverá ser fornecida pelo Cliente
      • LDAP DN Login:
        • É a string DN (Distinguished Name = Nome Distinto) de LOGIN para realizar a busca no Active Directory
        • Essa informação deverá ser fornecida pelo Cliente
        • A literal deverá fazer parte dessa string, pois ela identifica onde o login informado pelo usuario será substituído nela
          • Quando o cliente fornecer a STRING DN do Login, você deverá identificar qual o parametrô que se refere ao login de usuário e substituir por essa literal
      • Configuração dos Perfis:
        • Nessa seção serão listados todos os perfis atuais do PKPLUS.
        • Para cada perfil existente, deveremos realizar uma configuração, mesmo que repetida
        • Essas configurações serão utilizadas para vincular o perfil do usuario no Active Directory com os Perfis que possuímos cadastrados no sistema
        • Selecione um perfil no combo e clique em "Inserir Perfil"
          • Valores padrão para tomar como base serão inseridos.
          • Para alterar cada valor clique na tabela e altere
        • Perfil:
          • Nome do Perfil no PKPLUS que estamos configurando
          • Esse campo não é editável, você precisa selecionar no combo de perfis e clicar em "Inserir Perfil"
        • LDAP DN do Perfil:
          • É a String DN para identificar o Perfil de usuário no AD.
          • Essa informação deverá ser fornecida pelo Cliente
          • Ela deverá indicar qual a DN que é a referencia para atribuir que o usuário logado faz parte do perfil selecionado no PKPLUS
          • Normalmente ele deverá ser diferente para cada perfil do sistema
        • LDAP Filtro
          • É a informação de como iremos localizar o usuario logado na DN do grupo informada anteriormente
          • Essa informação deverá ser fornecida pelo Cliente
          • A literal deverá fazer parte dessa string, pois ela relacionará como procurar o usuario logado no grupo
          • Quando o cliente fornecer a STRING DN do Grupo e Filtro, você deverá identificar qual o parametrô que se refere ao login de usuário e substituir por essa literal
    4. Ao confirmar essas configurações serão suficientes para que o fluxo de Login no Caixa, Manager e Papafila sejam alterados conforme explicado na próxima seção

    Atenção

    Não esqueça de clicar no botão "CONFIRMAR" para Salvar as alterações. Caso contrário todas as alterações serão perdidas ao sair da tela. O botão "CANCELAR" retorna as configurações originais antes das mudanças

    Login com Case Insentive

    Atualmente o Sistema PKPLUS não faz distinção de maiusculas e minusculas, e isso pode ser impeditivo no login do AD do cliente. Para contornar essa situação foi criada uma chave avançada que deverá ser setada nas estações que realizam login:

    SECAO: SISTEMA

    CHAVE: LOGIN_CASE_SENSITIVE

    VALOR PADRÃO: 0 (alterar para 1)

    Fluxo do LOGIN

    Com a ativação e configuração do AD, o fluxo de Login do PKPLUS sofrerá mudanças que estão descritas a seguir:

    Atenção

    O Fluxo a seguir só deve ser considerado se o AD estiver marcado como ATIVO

    1. O Login SUPERUSER (WPSBRASIL) nunca tentará conectar no AD. Ou seja, esse usuario sempre se loga com o fluxo normal do sistema.
    2. O usuário deverá informar o usuário e senha normalmente, igual ele faz hoje nos sistemas.
      • O login e senha deverão ser os que o usuário possui no AD
    3. Ao informar o Login e Senha e confirmar, o sistema tentará realizar o login no AD configurado
    4. Caso Usuário e senha corretos
      • O Login no AD é realizado e o sistema tenta identificar qual o grupo (perfil) do usuario
      • Se conseguir encontrar o GRUPO/PERFIL:
        • o sistema salva ou atualiza na tabela users aquele usuario com o respectivo perfil e atribui uma senha provisória que somente o sistema tem conhecimento
        • Após isso o fluxo de login segue normalmente, ou seja, procura o usuario na tabela USUARIOS e usa a senha temporaria para validá-lo (como se fosse a senha digitada pelo usuario)
          • Fluxo normal de Login
      • Se NÃO conseguir encontrar o GRUPO/PERFIL:
        • Retorna um erro de Login pois o login foi realizado com sucesso no AD mas o usuario não tem relacionamento com nenhum PERFIL no PKPLUS
        • É necessário configurar o grupo que o usuário faz parte no AD com algum perfil do PKPLUS
        • O LOGIN NÃO CONTINUA
    5. Caso o Usuário Correto e a Senha incorreta:
      • Retorna erro de login, pois o usuario existe no AD mas a senha informada esta incorreta
      • O LOGIN NÃO CONTINUA
    6. Caso usuário informado não exista no AD
      • O sistema não retorna erro de imediato, ele tentará localizar o usuário na base interna do PKPLUS
      • Segue o processo de LOGIN normal, e se o usuario existir na nossa base, ele será logado no PKPLUS
