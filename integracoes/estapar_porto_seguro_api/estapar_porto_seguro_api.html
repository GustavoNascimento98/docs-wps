    Integração Estapar/Porto Seguro via API

    Informações sobre a integração com Estapar/Porto Seguro para obter validação do cliente para aplicação do desconto Porto Seguro Auto.

    Caixa e EPA consultarão a API Rest utilizando PPGateway.

    Versão mínima

    Versão mínima do sistema: 7.364.1

    Instalação PPGateway

    Consulte o documento Serviço PPGateway item 5.2 para consultar informações atualizadas de instalação e teste de funcionamento. Segue esta informação extraída do documento no dia 28/04/2021:

    Quote

    5.2 - Instalação do Serviço PPGateway

    A Forma de instalação do Serviço PPGateway segue o padrão de outros serviços que já rodam no Tomcat.

    O Tomcat deverá ser instalado na versão 7 no servidor da garagem. Normalmente basta descompactar o servidor em uma pasta com permissões completas de leitura e escrita, e antes de iniciar o Tomcat basta copiar o arquivo .WAR do do Serviço PPGateway para a pasta WebApps.

    Caso haja dúvidas na instalação do Tomcat, siga as Instruções de Instalação disponibilizada em outro documento.

    5.2.1 - Teste do Serviço

    Após iniciar o Tomcat e considerando que a porta padrão não foi alterada (8080), e o ip do servidor de banco de dados é 125.125.10.100 , basta abrir um browser qualquer e digitar:

    http://125.125.10.100:8080/servicoppgateway/servicos/test

    Onde:

    localhost: deverá ser alterado para o IP onde o serviço TomCat esta rodando

    servicoppgateway: é o nome do serviço dentro da pasta WebApps

    A resposta no browser deverá ser algo parecido com:

    {"Funcionando": true, "Versão do Serviço": "3.0.1", "Storage": "false", "Envio Automático Lote RPS": "false"}

    Configurações

    Configurando dados de acesso ao serviço Estapar/Porto Seguro

    As configurações devem ser realizadas através do Parking Plus Manager, menu Configurações → Monitor → Integração Porto Seguro:

    Deve-se configurar corretamente as informações da tela acima, conforme descrito na tabela abaixo:

    Campo Descrição
    Ativar Habilita ou não a integração Porto Seguro via API
    URL Base URL do serviço Estapar/Porto Seguro (fornecido pela Estapar)
    Usuário Usuário para autenticação no serviço Estapar/Porto Seguro (fornecido pela Estapar)
    Senha Senha para autenticação no serviço Estapar/Porto Seguro (fornecido pela Estapar)
    URL Gateway WPS URL do PPGateway WPS
    Cód. Garagem Código da garagem que será utilizado na utilização do serviço Estapar/Porto Seguro (fornecido pela Estapar)

    Dica

    Nos campos URL Base e URL Gateway Garagem, clique com o botão direito do mouse em cima do campo para abrir opção para preencher o campo com uma URL padrão sugerida.

    Configurando um desconto Porto Seguro

    PPC

    Deve-se adicionar um desconto na tela de tarifas - acessando pelo Parking Plus Manager → Configurações → Tarifador → Tarifas:

    Preencher a parte de descontos com o tipo "Porcentagem", 30 no campo "Valor" (30% de desconto) - consultar Estapar para definição deste valor - e escolher Cartão PORTO AUTO, nome sugerido do desconto "PORTO SEGURO" e ativar o desconto pelo campo "Estado" selecionando "Ativo".

    EPA

    Para a EPA, é necessário ativar a tela "Cartão Porto Seguro" em Parking Plus Manager → Configurações → EPA → Telas:

    Além de configurar a porcentagem do desconto na configuração avançada (Parking Plus Manager → Configurações → Monitor → Configurações Avançadas):

    Seção

    Chave

    Tipo

    Padrão

    #

    DESCONTO - EPA

    PORCENTAGEM DE DESCONTO PORTO AUTO

    Númerico

    0

    Descrição

    Determina o valor em "porcentagem" do desconto porto.

    Deve ser um número entre 0 e 100. Por Exemplo, o número 30 aplica um desconto de 30% sobre o valor liquido da transação.

    Tag3

    Para uso do desconto Porto via API com Tag3, os seguintes requisitos se aplicam:

    1. A Tag deve ser da marca1 Porto Seguro.
    2. A API deve autorizar a placa cadastrada no Tag. O Parking Plus não sabe se a placa cadastrada diverge da real.

    Sendo satisfeitos os requisitos acima, altere as seguintes configurações no PPM, no menu Configurações > Monitor > Serviço Tag Pag.Auto.

    Tip

    Atualmente apenas a ConectCar suporta estas configurações.

    Chave Descrição
    descporto_permite_label_text Texto informado pela OSA quando a Tag for da marca Porto Seguro. Pergunte para a OSA qual o valor informado em IsserLabel. Deixar este campo vazio desativa o desconto Porto Tag3.
    descporto_valor_percentual_0_a_100 Percentual de desconto, 0 = desativado. e.g. 30 = 30% de desconto sobre o valor liquido da transação.

    Aplicando desconto no Caixa

    Para aplicar o desconto no caixa, deve-se seguir o seguinte fluxo:

    1 - Após fornecer ticket ou placa para pagamento e/ou saída, acessar lista de descontos para escolha (tecla F1 para Caixa Dig. Placas e F8 para Caixa Barras):

    2 - Escolha o desconto configurado para cartão Porto Auto. Clique em "Confirmar".

    3 - Será solicitado para confirmar a placa do ticket ou fornecer a placa válida para o desconto Porto Auto:

    4 - Clique em Confirmar. Neste momento, será consultado o serviço Estapar/Porto Seguro.

    4.1 - Caso sucesso e placa for liberada, o desconto será aplicado:

    4.2 - Caso placa não liberada, desconto não será aplicado:

    4.3 - Caso ocorra problema para acessar o serviço Estapar/Porto Seguro, o sistema emitirá uma mensagem informando o problema e pedirá o número da Apólice, contido no Cartão Porto Seguro Auto, para realizar a validação offline:

    4.3.1 - Informe o número da apólice e, caso número válido, o desconto será aplicado:

    4.3.2 - Caso inválido, o desconto não será aplicado:

    5 - A partir daí, é só prosseguir para o pagamento.

    Aplicando desconto na EPA

    1 - Efetue a leitura do ticket na EPA, que irá exibir a tela para confirmação do desconto:

    2 - Escolha "Sim".

    3 - Será exibida tela para confirmar a placa para validação do desconto:

    4 - Escolha "Confirmar". Neste momento, será consultado o serviço Estapar/Porto Seguro.

    4.1 - Caso sucesso e placa for liberada, o desconto será aplicado:

    4.2 - Caso placa não liberada, desconto não será aplicado:

    4.3 - Caso ocorra problema para acessar o serviço Estapar/Porto Seguro, o sistema pedirá o número da Apólice, contido no Cartão Porto Seguro Auto, para realizar a validação offline:

    4.3.1 - Informe o número da apólice e, caso número válido, o desconto será aplicado:

    4.3.2 - Caso inválido, o desconto não será aplicado:

    5 - A partir daí, é só prosseguir para o pagamento.

    Aplicando descontos no Tag3

    Se as configurações foram feitas corretamente, a aplicação será automática.

    Refira-se aos registros de tentativas (abaixo) e/ou logs do Parking Plus para identificar problemas, caso ocorram.

    Registros de todas as tentativas de consulta da integração

    Existe o registro em tabela específica no BD do Sistema Parking Plus WPS de todas as tentativas de acesso à API Estapar/Porto Seguro para eventual rastreio de problemas e auditorias.

    Nome da tabela: porto_seguro_api_log

    Nome do campo Descrição
    id Sequencial da tabela.
    datahora Data/hora do momento da tentativa de acesso à API Estapar/Porto Seguro.
    metodo Nome do método da API Estapar/Porto Seguro onde ocorreu o problema.
    response_code Código da resposta da API Estapar/Porto Seguro.
    response_text Texto da resposta da API Estapar/Porto Seguro.
    response_body Corpo da resposta da API Estapar/Porto Seguro.
    token Token utilizado no envio das informações de pagamento.
    placa Placa enviada à API Estapar/Porto Seguro.
    analise_wps Descrição da WPS referente ao problema.
    ticket Número do ticket enviado nas informações de pagamento.
    success Indica se houve sucesso no método Authenticate e se placa foi validada no método ValidateClient.
    expirationdate Data/hora de expiração do token recebido no método Authenticate.
    returnmessage Mensagem retornada nos métodos Authenticate e ValidateClient.
    canal Conteúdo do campo "Canal" retornado no método ValidateClient.
    validate_timestamp Conteúdo do campo "TimeStamp" retornado no método ValidateClient.
    validate_protocol Conteúdo do campo "Protocol" retornado no método ValidateClient.
    validate_valid Conteúdo do campo "Valid" retornado no método ValidateClient. Indica se placa foi validada pela Porto Seguro.

    1. O Parking Plus identifica a marca do Tag segundo uma identificação adicional enviada pela OSA. O Tag continua sendo identificado apenas pela OSA. A marca serve apenas para o desconto Porto. Inicialmente apenas a ConectCar utiliza esta identificação. ↩
