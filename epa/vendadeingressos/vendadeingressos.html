    Venda de Ingressos na EPA

    Descrição

    A funcionalidade de Venda de Ingressos na EPA é realizada via integração com APIs externas. Neste caso, são utilizadas APIs fornecidas através do cliente SENAP e seus parceiros/fornecedores ANCAR e PWI.

    A EPA consultará as APIs utilizando o serviço PPGateway WPS.

    PWI fornece a API para disponibilização de produtos e efetivação da venda.

    ANCAR fornece a API para consulta e confirmação de uso de QrCode (cupom) emitido por seu app para o fornecimento de itens de “promoções” ao usuário.

    Configurações necessárias

    Instalação PPGateway

    Consulte o documento Serviço PPGateway item 5.2 para consultar informações atualizadas de instalação e teste de funcionamento. Segue esta informação extraída do documento no dia 28/04/2021:

    Quote

    5.2 - Instalação do Serviço PPGateway

    A Forma de instalação do Serviço PPGateway segue o padrão de outros serviços que já rodam no Tomcat.

    O Tomcat deverá ser instalado na versão 7 no servidor da garagem. Normalmente basta descompactar o servidor em uma pasta com permissões completas de leitura e escrita, e antes de iniciar o Tomcat basta copiar o arquivo .WAR do do Serviço PPGateway para a pasta WebApps.

    Caso haja dúvidas na instalação do Tomcat, siga as Instruções de Instalação disponibilizada em outro documento.

    5.2.1 - Teste do Serviço

    Após iniciar o Tomcat e considerando que a porta padrão não foi alterada (8080), e o ip do servidor de banco de dados é 125.125.10.100 , basta abrir um browser qualquer e digitar:

    http://125.125.10.100:8080/servicoppgateway/servicos/test

    Onde:

    localhost: deverá ser alterado para o IP onde o serviço TomCat esta rodando

    servicoppgateway: é o nome do serviço dentro da pasta WebApps

    A resposta no browser deverá ser algo parecido com:

    {"Funcionando": true, "Versão do Serviço": "3.0.1", "Storage": "false", "Envio Automático Lote RPS": "false"}

    Configurações produtos de promoções

    Para o processo de validação do cupom (QrCode) é preciso ter os produtos de promoção cadastrados que deverão ser listados após uma validação com sucesso. Para cadastrá-los, no PPManager, vá em menu Configurações → Monitor → Integração Venda de Ingressos → Cadastro de Produtos:

    Descrição dos campos:

    Nome Descrição
    Id Produto Identificação do produto, que deve ser igual ao fornecido pela API de disponibilização de produtos.
    Descrição Descrição do produto - preenchimento livre
    Validade Data de Validade para uso do produto - preenchimento livre

    Baixa de pagamentos parciais ou a mais em dinheiro

    Nos pagamentos com dinheiro, caso o valor recebido pela EPA não seja o valor exato da compra, a EPA devolve o troco (no caso de receber valor maior que o total da compra).

    Nos casos de o valor recebido ser menor que o valor total da compra ou a EPA não conseguir devolver todo o troco, será emitido um recibo para que o cliente se dirija a um caixa e seja reembolsado.

    Atenção

    No pagamento com valor menor do que o valor total da compra, a venda não é efetivada.

    No Caixa, há a possibilidade de "dar baixa" neste registro para que ele não seja reembolsado novamente. Pelo menu Operações - Venda de Ingressos - Baixa de Pagamento Parcial, será exibida a tela abaixo:

    Nela, é possível filtrar o registro que se pretende "dar baixa" pelo período, número da transação (consta no recibo que o cliente retira na EPA) ou documento que o cliente utilizou no momento da compra.

    Para "dar baixa" no registro, selecione ele na lista e clique em "Baixar item selecionado".
